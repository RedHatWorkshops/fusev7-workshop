Target Audience

  • The Citizen Developer

    • Does not code

    • Strong in business requirements knowledge

    • Needs a tool like Fuse Online, which does not require coding skills to accomplish integrations.

Technical Features

  • Generic API Connector

    • Allows any API to be imported as a connector

  • Step Extensions

    • Ability to natively extend the platform by adding steps

  • AMQ connector

    • Compatible with AMQ xPaaS images

  • Database connector

    • Accepts user-defined SQL

  • Amazon S3 connector

    • Cloud-based file transfer

  • Import and export facility for integrations

    • Facilitate transfer of integrations between local and Fuse Online environments

Terminology

  • Integration

    • Integration project involving multiple datasources or applications.

  • Step

    • A task that performs data manipulation, retrieval or persistence.

    • Each integration typically comprises of several of these.

  • Connector

    • A building block for a connection.

  • Connection

    • A link to either a datasource or an application.

Pre-packaged software

  • PostgresDB and AMQ are embedded by default

  • Default Database - PostgresDB

    • Provide necessary persistent data storage

    • Suitable for user scenarios requiring data management

    • users without data requirements can safely ignore the database

    • external databases (MariaDB, MySQL, IBM DB2, Microsoft SQL Server) can be integrated through

      • JDBC drivers packaged as technical extensions and

      • pre-packaged SQL component

  • Default Message Broker - Red Hat AMQ

    • Provides reliable and asynchronous messaging support

    • Addreses use cases which require inter-systems communication

    • users without reliable messaging requirements can safely ignore the Red Hat AMQ broker

Competitive Analysis

  • Mulesoft
    • Proprietary integration technology
    • Uncertain future after acquisition by Salesforce
    • Separate licensing for integration messaging support
    • Size of cluster limited to eight server instances
    • Public Cloud offering only
    • No open source licensing, only proprietary licensing
    • Cluster deletion and recreation mandatory for upgrades
    • No out-of-the-box Enterprise Integration Patterns
    • Only local transaction support for JMS, JDBC and VM queues
    • Unable to scale and support large population (>100) of microservices
    • No support for AMQP, a popular open-standards messaging service
    • Inflexible tooling strategy - limited to Mule Studio environment
    • No bundled messaging service
    • API monetization module is absent - has to be custom developed by end-user

Competitive Analysis

  • Talend
    • Additional subscription costs for critical features:
      • Enterprise Integration Patterns
      • Load-balancing
      • Failover
      • Monitoring
      • Administration
    • Significantly higher in total cost of ownership due to multiple types of subscription costs
    • No Complex Event Processing
    • Only partially open-source
    • Most supported connectors are database-centric
    • Few supported connectors suited for service integration
    • No Platform-as-a-Service offering
    • Small ecosystem community
    • Fewer contributions to the upsteam project vs Red Hat's contribution to Apache Camel
